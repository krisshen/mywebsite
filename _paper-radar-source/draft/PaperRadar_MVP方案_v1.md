# AI 论文自动情报系统 · 完整 MVP 方案

> **产品代号**：PaperRadar  
> **定位**：每天为你筛出 5 篇真正值得看的 AI 论文，并让任何人都能读懂

## 二、技术栈建议

| 层级 | 技术选型 | 理由 |
|------|---------|------|
| 前端框架 | Next.js 14 (App Router) | SSG/ISR 支持，SEO 友好，部署简单 |
| 样式 | Tailwind CSS + shadcn/ui | 快速出图，组件齐全 |
| 数据存储 | Vercel KV（Redis）+ JSON 文件 | MVP 阶段零成本 |
| AI 调用 | Anthropic Claude API (claude-sonnet-4) | 摘要质量最佳 |
| 定时任务 | Vercel Cron Jobs | 无需额外服务器 |
| 部署 | Vercel | 一键部署，免费额度足够 |
| RSS 解析 | rss-parser (npm) | 轻量可靠 |
| 监控 | Vercel Analytics | 内置，无额外配置 |

**MVP 阶段总成本估算**：$0–$20/月（Claude API 按量计费）

## 三、数据流说明

```
每日 08:00 UTC 触发 Cron Job
        │
        ▼
1. 抓取 RSS
   GET https://rss.arxiv.org/rss/cs.AI
   GET https://rss.arxiv.org/rss/cs.CL  
   GET https://rss.arxiv.org/rss/cs.LG
   → 合并去重，得到 ~60 篇候选
        │
        ▼
2. 过滤评分
   - 去除非英文标题
   - 去除重复 arXiv ID
   - 基于关键词打分（LLM/Agent/Multimodal/RAG/等）
   - 取前 20 篇进入处理队列
        │
        ▼
3. Prompt Pipeline（串行处理，控制 API 成本）
   每篇论文调用 1 次 Claude API，生成完整 JSON
   Input：标题 + abstract（arXiv RSS 自带）
   Output：见 JSON Schema（第四节）
        │
        ▼
4. 写入存储
   - Vercel KV: SET "papers:YYYY-MM-DD" → JSON Array
   - 保留最近 30 天数据
        │
        ▼
5. 前端读取
   - 首页：ISR 每小时重新验证
   - 论文详情：静态生成 + 客户端 hydrate
   - 难度切换：纯前端状态，无需 API 调用
```

## 五、Prompt 模板（分级）

### Master Prompt（单次调用生成全部字段）

```
你是一位 AI 研究领域的内容分析专家，擅长将学术论文转化为不同层次的易读内容。

以下是一篇 arXiv 论文的信息：

标题：{{title}}
作者：{{authors}}
摘要（原文）：{{abstract}}
分类：{{categories}}

请严格按照以下 JSON 格式输出分析结果，不要有任何额外说明：

{
  "summaries": {
    "expert": "【专业版】保留所有技术术语（如 attention mechanism、RLHF、LoRA 等），面向 AI 研究者，精准描述方法论和实验结果。200字以内。",
    "general": "【通用版】用类比解释核心思路，避免缩写，面向有编程基础的工程师。例如：'就像给模型装了一个...的过滤器'。150字以内。",
    "lazy": "【懒人版】一句话说清楚：这篇论文做了什么、有什么用。不超过50字。必须包含具体的应用场景。"
  },
  "analysis": {
    "innovation_points": [
      "创新点1：具体说明，不超过30字",
      "创新点2：具体说明，不超过30字"
    ],
    "business_potential": "从产品/商业角度分析：谁可以用这个技术？能解决什么实际问题？预估商业价值。100字以内。",
    "risks": [
      "风险1：技术/伦理/实用性层面的具体风险"
    ],
    "trend_tags": ["从预设列表选1-3个标签"],
    "recommendation_score": 1到5的整数,
    "recommendation_reason": "值得关注因为... / 暂时观望因为...，30字以内"
  }
}

输出要求：
- 严格 JSON 格式，无 markdown 代码块
- 中文输出
- 不要编造论文中没有的内容
- trend_tags 只能从以下选择：LLM、Agent、RAG、Multimodal、Reasoning、RLHF、Efficiency、Safety、Vision、Code、Speech、Graph、Diffusion、FineTuning、Benchmark、其他
```

### 过滤评分 Prompt（批量，用于筛选 20 篇）

```
以下是今日抓取的 AI 论文列表（标题+摘要前100字），请从产品价值和普通用户兴趣角度打分。

{{papers_list}}

对每篇论文给出 1-10 的分数，评分标准：
- 有实用产品/应用价值 +3分
- 涉及大模型/Agent/多模态热门方向 +2分
- 结论清晰易理解 +2分
- 纯理论/数学推导为主 -3分
- 评测/数据集类论文 -1分

返回 JSON：[{"id": "arxiv_id", "score": 8}, ...]
按分数降序排列，只返回前20条。
```

**【第一段：钩子 0:00-0:05】**

---

**【第二段：问题建立 0:05-1:00】**
> 三幕场景快速切：  
> 场景 A："研究生模式"——打开 PDF，第一段就懵了  
> 场景 B："工程师模式"——复制摘要给 ChatGPT 问了半小时  
> 场景 C："普通人模式"——直接关掉，刷抖音去了  
>   
> 旁白：*"你属于哪一种？我猜都是B和C。"*

**【第三段：三种读论文方式 1:00-3:30】**

> 展示：PDF原文，20页，晦涩术语

**方式三：PaperRadar（2:30-3:30）**
> 展示：打开网站，今日5篇已经等你了

**【第四段：系统演示 3:30-6:00】**

**【第五段：降智按钮特写 6:00-7:30】**

## 十、风险分析

### 技术风险

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| arXiv RSS 结构变更 | 低 | 高 | 监控 RSS 格式，备用官方 API |
| Claude API 超时/限流 | 中 | 中 | 串行处理控制速率，失败重试机制 |
| Vercel KV 超配额 | 低 | 中 | 本地 JSON 文件作为备用方案 |
| 摘要质量不稳定 | 中 | 高 | Few-shot 示例优化 Prompt，人工审核前5篇 |

### 产品风险

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| 用户留存低 | 中 | 高 | 邮件订阅 + B站更新驱动回流 |
| 论文摘要有误导性 | 中 | 中 | 始终提供原文链接，注明"AI生成摘要" |
| 版权/引用争议 | 低 | 中 | arXiv 为开放平台，显示摘要符合 fair use |
| 竞品（如 papers with code）挤压 | 中 | 中 | 差异化：中文 + 降智按钮 + 视频联动 |

### API 成本估算

```
每日处理 20 篇论文
每篇 Prompt ≈ 800 tokens input + 600 tokens output
每日消耗 ≈ 28,000 tokens
claude-sonnet-4 成本 ≈ $0.04/日 ≈ $1.2/月
```
→ **MVP 阶段 API 成本极低，无需担心**
